### 2. 坐标系转换
严格遵循 ROS2 标准坐标系（头颈部正前方向）：
- **yaw（偏航角）**: 左右转头，正值=左转，负值=右转
- **pitch（俯仰角）**: 上下点头，正值=低头，负值=抬头
- **roll（翻滚角）**: 左右歪头，正值=向左歪，负值=向右歪

### 3. 场景转换

#### 场景1: 用户移动位置时的视线跟踪
- **原场景**: `user_position_tracking` (包含底盘控制)
- **新场景**: 仅头部控制
- **参数**: pitch=30°(低头), yaw=30°(左转), roll=0°
- **转换**: 移除底盘静止控制，保留头部动作

#### 场景2: 头部电机回归0位
- **原场景**: `head_reset_to_zero`
- **新场景**: 4自由度归零
- **参数**: yaw=0°, pitch=0°, roll=0°
- **转换**: 新增roll轴明确归零

#### 场景3: 左右摆头观察
- **原场景**: `move_forward_with_head_sweep` (包含底盘前进)
- **新场景**: 仅头部摆动序列
- **参数**: 左转80° → 右转80° → 回中0°
- **转换**: 移除底盘前进控制，保留头部摆动

#### 场景10: 4自由度头颈手动控制
- **功能**: 交互式输入 yaw/pitch/roll 角度
- **特点**: 支持自定义任意角度组合
- **转换**: 度数自动转换为弧度

### 4. 接口适配

#### 4自由度控制接口结构
```python
{
    "type": "set_four_combine_motor_control",
    "params": {
        "control_yaw": True,      # 是否控制偏航
        "yaw_angle": 0.524,       # 偏航角度（弧度）
        "control_pitch": True,    # 是否控制俯仰
        "pitch_angle": 0.524,     # 俯仰角度（弧度）
        "control_roll": False,    # 是否控制翻滚
        "roll_angle": 0.0,        # 翻滚角度（弧度）
        "speed_level": 1          # 速度档位 (0/1/2)
    }
}
```

#### 与原接口的区别
- **原接口**: `set_combine_motor_control` (2自由度: pitch + yaw)
- **新接口**: `set_four_combine_motor_control` (4自由度: yaw + roll + pitch + 底盘)
- **本脚本**: 仅使用头颈3自由度 (yaw + roll + pitch)，不使用底盘控制
